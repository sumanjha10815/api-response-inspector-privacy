Privacy Policy for API Response Inspector

Last Updated: January 1, 2025

1. Introduction

API Response Inspector (“the Extension”, “we”, “us”, or “our”) is a Chrome DevTools extension that allows developers to inspect, analyze, and export API network requests and responses generated by web pages they explicitly choose to inspect.

This Privacy Policy explains how the Extension accesses, uses, stores, and protects data in accordance with the Chrome Web Store Developer Program Policies, including Google’s User Data and Limited Use requirements.

2. Data Access and Collection

2.1 Data Accessed by the Extension

When the Extension is enabled within Chrome DevTools, it accesses network traffic from the currently inspected tab only, including:

  • URLs of API requests
  • HTTP request methods
  • Request headers (including cookies and authorization headers)
  • Request bodies
  • Response headers
  • Response bodies
  • HTTP status codes
  • Network timing information

The Extension also accesses cookies and related request metadata as part of Chrome’s DevTools network inspection APIs.

2.2 Sensitive and Personal Data Disclosure

The Extension does not intentionally collect personal data. However, because it inspects raw API traffic, captured data may include sensitive or personal information depending on the behavior of the inspected website, such as:

  • Authentication tokens and credentials
  • Personally identifiable information (PII)
  • Financial or payment-related data
  • Health-related data
  • Location data
  • Personal communications
  • Any other data transmitted via API calls

All accessed data originates exclusively from websites chosen by the user for inspection.

3. Purpose of Data Use

Data accessed by the Extension is used solely and exclusively to:

  • Display API request and response details in Chrome DevTools
  • Enable debugging and analysis of network activity
  • Allow users to generate and download HAR (HTTP Archive) files
  • Support scheduled capture jobs configured by the user

The Extension does not use collected data for any other purpose.

4. Data Storage and Transmission

4.1 Local Processing and Storage

  • All data is processed and stored locally on the user’s device
  • Storage uses Chrome’s chrome.storage.local API
  • Data is never transmitted off the user’s device
  • No remote servers, cloud storage, or external services are involved

4.2 No Data Sharing or Selling

  • We do not collect, transmit, sell, or share user data
  • We do not use analytics, telemetry, advertising, or tracking technologies
  • We do not share data with third parties under any circumstances

5. Data Retention and Deletion

  • Network data is retained in memory only while the DevTools panel is open
  • Scheduled capture jobs and HAR files remain stored locally until deleted by the user

Users may delete all stored data at any time by:

  • Clearing captured requests
  • Deleting scheduled jobs or HAR files
  • Uninstalling the Extension

Uninstalling the Extension removes all locally stored data.

6. User Control and Consent

  • The Extension only operates when explicitly enabled by the user in Chrome DevTools
  • Users have full control over what data is captured
  • Users can stop data collection at any time by closing DevTools or disabling the Extension

The Extension does not collect data in the background without user awareness.

7. Security Practices

  • All data remains on the user’s local device
  • No external network communication is performed
  • The Extension relies solely on Chrome’s built-in DevTools APIs
  • Reasonable technical measures are used to prevent unauthorized access within the browser environment

8. Compliance With Chrome Web Store Policies

API Response Inspector complies with the Chrome Web Store policies by:

  • Accessing user data only for a single, clearly disclosed purpose
  • Processing all data locally
  • Not transferring data to external servers
  • Not using data for advertising, analytics, or profiling
  • Not selling or sharing user data
  • Providing users full control over data collection and deletion

9. Changes to This Policy

We may update this Privacy Policy from time to time. Changes will be reflected by updating the Last Updated date at the top of this document.

Continued use of the Extension after changes constitutes acceptance of the updated Privacy Policy.
